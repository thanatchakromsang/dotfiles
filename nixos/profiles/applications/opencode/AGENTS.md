# Global Software Development

## Commands
Build/test before push. Run linter if available. Validate configs with dry-run.

## Code Style
Match existing patterns. 2-space indent. No comments unless asked.

## Git
Commit only when requested. Never force push main. Use conventional commits: type(scope): description. Types: feat, fix, docs, style, refactor, perf, test, chore, revert.

## Security
Never commit secrets. Use env vars for sensitive data. Validate inputs. Log events never secrets.

## Memory
- MEMORY.md: long-term lessons
- memory/YYYY-MM-DD.md: daily logs
- Read today/yesterday at session start

## AI Best Practices
DO: Follow patterns, write tests, handle errors, validate inputs
DON'T: Skip validation, ignore errors, over-engineer, hardcode paths
